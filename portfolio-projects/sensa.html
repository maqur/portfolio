SenSa

Overview

For our third year Embedded Systems coursework we had to develop an IoT product with the provided sensors. My group was given an Adafruit Huzzah Feather development board which has an ESP8266 WiFi microcontroller for processing and communication, and an HMC5883L magnetometer sensor.

Product Description

We decided to build a smart home solution in which the sensor will be attached to a wire for detecting the flow of current. This can be used to record the time for the which the appliance was turned on and if the voltage is known the power consumption can also be approximated. Several sensors attached to appliances around the house will allow the power usage of the whole house to be analyzed. Although machine learning was not implemented it can be used in the future to make suggestions on how the power usage can be reduced. Check out the SenSa website for more details on the product.

Implementation

The huzzah feather board was programmed using micro python and the data from sensor was obtained using the I2C protocol. The raw data from the sensor was converted on board to the units gauss and thresholds were used to detect when the device was turned on and off. MQTT protocol was used to communicate with the server. When the development board is turned on it subscribes to a topic to get the current time and when it detects that the connected appliance is turned on or off it records the time and publishes the information on a specified topic. The server gets this information using a subscriber and records it in a database. This information is analyzed and presented in user readable way when needed.

Outcome

During this project I learnt to use the MQTT server and the I2C protocols both of which I was unfamiliar with before. I was also able improve my skill in Python programming language.
